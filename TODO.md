# TODO List - Model Switcher 拓展计划

> 本项目坚守四大原则：轻量极简、用户无感、便携切换、完全本地安全。  
> 所有新增功能必须通过“原则测试”：不增加常驻资源消耗、不引入静默联网、不复杂化代码结构。

## 0. 跨平台支持（核心拓展）

- [ ] **统一启动/停止机制**  
       为 macOS（`launchd`/`plist`）和 Linux（`systemd user unit` 或 `.desktop autostart`）提供与现有 `start-silent.vbs` 等效的静默后台方案。
- [ ] **无控制台运行适配**  
       确保 `launcher.js` 和 `server.js` 在三大平台上均能完全脱离终端窗口运行（利用 `pkg` 打包或原生脚本）。

## 1. 可扩展架构（多 API / 多工具）

- [ ] **插件式 Provider 适配器**  
       抽象出统一的 `ProviderAdapter` 接口（如 `switch(config)`、`getFields()`），将 Claude、OpenCode 等工具的支持实现为独立模块，放入 `providers/` 目录。
- [ ] **动态表单渲染**  
       `index.html` 不再硬编码字段，而是从 Provider 模块提供的元数据动态生成界面，新增 Provider 无需改动前端代码。
- [ ] **通用配置模型**  
       `config.json` 采用扁平化的 Provider-Profile 结构，一个配置文件描述所有已添加的模型/端点组合，格式清晰且易于手动编辑。

## 2. 功能增强（安全本地优先）

- [ ] **配置导入/导出**  
       支持将当前所有配置导出为加密可选（仅本地口令）的 JSON 文件，并可一键导入。彻底替代任何云同步方案，保持完全本地。
- [ ] **API Key 连通性测试**  
       在管理界面提供“测试连接”按钮，向对应 Provider 的校验端点发起一次明确告知的请求，验证 Key 的有效性（无静默联网）。
- [ ] **更多 Provider 预设模板**  
       内置 Gemini、Mistral、Groq 等主流 API 的默认参数（端点、模型名格式），让用户开箱即用，同时保持模板本身为纯配置文件，不内置任何密钥。

## 3. 体验优化（无感 & 便携）

- [ ] **轻量级系统托盘（可选）**  
       使用极简原生方案（如 Windows 上基于 `Shell_NotifyIcon` 的轻量托盘、macOS 的 `menubar`）提供快速切换入口，**必须保持空闲时零 CPU 占用**，且与现有按需启动机制共存，不影响后台自动休眠。
- [ ] **暗黑/明亮主题**  
       Web 管理界面增加主题切换，记忆用户偏好（本地 `localStorage`），不依赖外部资源。
- [ ] **工具配置向导**  
       首次添加配置时，提供分步向导，帮助用户理解所需字段，降低使用门槛。

## 4. 文档与社区

- [ ] **开发者文档**  
       编写 `server.js` API 接口文档，说明如何通过 HTTP 管理配置，方便其他脚本调用。
- [ ] **Provider 开发指南**  
       提供“如何添加一个新工具支持”的 Step-by-step 教程，只需复制模板、实现 3 个函数即可完成接入。
- [ ] **安全审计说明**  
       在 README 中明确列出所有网络行为（仅打开本地 Web 服务、用户手动触发的连通性测试），消除安全疑虑。
