## 阶段十：MCP 服务器管理

### 10.1 MCP 服务器统一管理

**目标**：统一管理 Claude Code 的 MCP 服务器配置，提供可视化卡片界面替代手动编辑 JSON。

**范围**：
- `providers/claude.js` 新增 `getMcpConfig()` / `writeMcpConfig()` 方法
- `server.js` 新增 4 个 MCP API 端点
- `index.html` 侧边栏新增"MCP 管理"导航项，新增 MCP 管理页面

**配置格式**：
Claude Code 的 MCP 配置存储在 `~/.claude/settings.json` 的 `mcpServers` 字段中：
```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"],
      "env": { "KEY": "value" }
    }
  }
}
```

**后端 API 设计**：

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET /api/mcp/claude` | GET | 读取 Claude Code 的 mcpServers 配置 |
| `POST /api/mcp/claude` | POST | 添加一个 MCP 服务器（body: `{name, command, args, env}`） |
| `PUT /api/mcp/claude/:name` | PUT | 编辑指定 MCP 服务器 |
| `DELETE /api/mcp/claude/:name` | DELETE | 删除指定 MCP 服务器 |

**适配器接口扩展**：
- `claude.js` 新增 `getMcpConfig()` — 读取 `settings.json` 的 `mcpServers`，返回对象
- `claude.js` 新增 `writeMcpConfig(mcpServers)` — 写入整个 `mcpServers` 对象（原子写入 + 校验）
- 其他适配器暂不实现，server.js 中对不支持 MCP 的工具返回 501

**前端页面设计**：
- 侧边栏导航最下方（footer 上方）新增"MCP 管理"项，图标用 `puzzle`
- 点击后内容区切换为 MCP 管理页面
- 卡片列表：每张卡片显示 name、command、args 摘要（截断）、env 数量指示
- 卡片操作：编辑按钮、删除按钮
- 页面顶部"添加 MCP 服务器"按钮，点击弹出添加弹窗
- 添加/编辑弹窗字段：名称（text）、命令（text，如 npx）、参数（textarea，每行一个）、环境变量（textarea，JSON 格式 `{KEY: VALUE}`）
- 支持 JSON 导入：弹窗内提供"粘贴 JSON 导入"区域，粘贴完整 MCP 配置 JSON 自动填充字段
- 删除前弹出确认对话框

**改动量**：约 60 行后端 + 30 行适配器 + 120 行前端

### 10.2 MCP 配置双向同步

**目标**：从 Claude Code 的 live 配置文件中读取已有 MCP 服务器，展示在面板中；面板修改后回写到配置文件。

**范围**：已在 10.1 的 `getMcpConfig()` / `writeMcpConfig()` 中实现，无需额外工作。

**同步机制**：
- 页面加载时调用 `GET /api/mcp/claude` 读取 live 配置
- 每次增删改操作后立即调用对应的写入 API
- `writeMcpConfig()` 写入时采用原子写入，保持与现有配置安全网一致

**改动量**：0 行（已包含在 10.1 中）

## 阶段十一：UI/UX 增强

### 11.1 Provider 搜索过滤

**目标**：在配置列表顶部新增搜索框，按名称/模型/Base URL 实时过滤。

**范围**：`index.html`

**设计**：
- 配置列表卡片顶部新增搜索输入框，带搜索图标（`search` Lucide icon）
- 输入后 200ms 防抖，过滤 `configs.filter(c => c.type === currentTool)` 的结果
- 过滤逻辑：名称、模型、Base URL 任一字段包含搜索词（不区分大小写）
- 搜索框为空时显示全部配置
- 空结果时显示"未找到匹配的配置"提示

**改动量**：约 25 行 JS + 10 行 CSS

### 11.2 批量连通性测试

**目标**：一键并行测试当前工具下所有已配置 Provider 的连通性。

**范围**：`server.js` + `index.html`

**后端**：
- 新增 `POST /api/test-all/:tool` 端点
- 读取 config.json 中该工具的所有配置
- 并行调用已有的测试逻辑（复用 `testers` 和 `providers.verify`）
- 返回每个配置的测试结果数组

**前端**：
- content-header 区域（工具标题旁）新增"全部测试"按钮
- 点击后：
  - 所有卡片的测试按钮进入 loading 状态
  - 并行调用 `POST /api/test/:id`（复用现有端点，无需新增后端端点）
  - 每个测试完成后立即更新对应卡片的状态图标
  - 全部完成后顶部显示统计摘要（如"3/5 通过"）
- 测试按钮图标变化：loading → 成功（绿色 check-circle）/ 失败（红色 x-circle）
- 结果 3 秒后恢复为默认图标

**改动量**：约 25 行 JS（前端并行调用逻辑）

> 注：为保持轻量，批量测试直接复用现有 `POST /api/test/:id` 端点在前端并行调用，不需要新增后端端点。

### 11.3 键盘快捷键

**目标**：提升操作效率，支持常用快捷键。

**范围**：`index.html`

**快捷键定义**：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+K` / `Cmd+K` | 聚焦搜索框 | 需要 `preventDefault` 阻止浏览器默认行为 |
| `Escape` | 关闭当前弹窗/对话框 | 优先级：wizardModal > editModal > 其他 |
| `Ctrl+N` / `Cmd+N` | 打开配置向导 | 需要 `preventDefault` |

**实现**：`document.addEventListener("keydown", handler)` 统一处理

**改动量**：约 20 行 JS

### 11.4 Provider 状态概览

**目标**：在 Provider 卡片上直观显示连接状态，用户打开页面即可了解各配置的健康状况。

**范围**：`providers/claude.js`（及其他适配器）写入后校验、`server.js` 测试端点、`index.html` 前端展示

**数据模型**：
config.json 每条配置新增字段：
```json
{
  "id": "1234567890",
  "name": "DeepSeek",
  "type": "claude",
  "...": "...",
  "lastTestStatus": "ok",
  "lastTestTime": "2026-05-29T16:00:00.000Z",
  "lastTestElapsed": 342
}
```

- `lastTestStatus`：`"ok"` / `"fail"` / `null`（未测试或编辑后重置）
- `lastTestTime`：ISO 8601 时间戳
- `lastTestElapsed`：响应时间（毫秒）

**状态更新时机**：
- 测试完成后（`POST /api/test/:id` 成功/失败时）写入 config.json 对应配置的三个字段
- 编辑配置保存时（`PUT /api/configs/:id`）将三个字段重置为 null

**前端展示**：
- 卡片名称左侧新增状态圆点：
  - 绿色（`lastTestStatus === "ok"`）
  - 红色（`lastTestStatus === "fail"`）
  - 黄色（`lastTestStatus === null`，未测试）
- Hover 圆点显示 tooltip：最后测试时间 + 响应时间（如"3 秒前 · 342ms"）
- 首页顶部（配置列表卡片标题旁）新增统计摘要：配置总数、上次测试通过数

**改动量**：约 20 行后端（测试端点写入状态）+ 30 行前端（圆点 + tooltip + 统计）
