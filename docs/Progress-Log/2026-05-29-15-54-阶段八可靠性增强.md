## 本次开发进度

### 8.1 配置文件原子写入

- 在 providers/utils.js 新增 atomicWrite 和 writeJsonAtomic 函数
- atomicWrite 采用 try-rename 优先策略，失败时 fallback 到 delete+rename（Windows 兼容）
- 临时文件名使用 Date.now() + crypto.randomBytes 保证唯一性
- 替换 server.js 中 6 处 writeJson 调用为 writeJsonAtomic
- 替换 claude/opencode/continue 适配器中的 writeJson 为 writeJsonAtomic
- 替换 codex 适配器中 3 处 fs.writeFileSync 为 atomicWrite / writeJsonAtomic
- 替换 aider 适配器中 writeConfig 的 fs.writeFileSync 为 atomicWrite
- 清除 4 个适配器中的本地 writeJson 冗余函数

### 8.2 配置文件自动备份

- 在 server.js 新增 backupFile 函数和 BACKUP_DIR/MAX_BACKUPS 常量
- 在 DELETE /api/configs/:id、POST /api/import、POST /api/switch/:id 三个端点的写入前调用 backupFile
- 新增/编辑配置不触发备份（非破坏性操作）

### 8.3 Claude Code 配置项补全

- claude.js switch 方法支持 apiKeyField 参数选择 ANTHROPIC_API_KEY 或 ANTHROPIC_AUTH_TOKEN

- getFields 新增 apiKeyField（select）和 apiTimeoutMs（text）字段
- index.html 编辑弹窗新增认证方式和超时输入框
