## 阶段二：功能增强（新增端点，不改现有逻辑）

目标：在 server.js 中新增少量 API 端点，前端新增对应 UI。

### 2.1 配置导入/导出

- **状态**：已结束
- **范围**：`server.js` 新增 2 个端点 + `index.html` 新增按钮
- **内容**：
  - `GET /api/export`：导出当前 config.json 为下载文件
  - `POST /api/import`：上传 JSON 文件覆盖配置（需确认提示）
  - 加密可选：用 `crypto.scrypt` + 用户口令，仅本地解密
- **改动量**：约 60 行后端 + 40 行前端
- **验收标准**：导出的文件可手动编辑后重新导入，配置完整恢复

### 2.2 API Key 连通性测试

- **状态**：已结束
- **范围**：`server.js` 新增 1 个端点 + `index.html` 新增"测试"按钮
- **内容**：
  - `POST /api/test/:id`：向对应 Provider 发一次轻量请求验证 Key 有效性
  - 仅用户点击触发，不自动调用
  - 返回连通状态 + 响应时间
- **改动量**：约 50 行后端 + 20 行前端
- **验收标准**：有效 Key 显示成功，无效 Key 显示具体错误

### 2.3 工具配置向导

- **状态**：已结束
- **范围**：仅 `index.html`
- **内容**：首次添加配置时的分步引导（选择 Provider → 填写 Key → 确认模型 → 完成）
- **改动量**：约 100 行前端
- **验收标准**：新用户可通过向导完成首次配置，老用户可跳过

- **验收标准**：macOS 上双击即可后台启动，重启后自动运行

### 3.2 Linux 启动脚本

- **状态**：已结束

- **改动量**：约 30 行 shell 脚本
- **验收标准**：Linux 上 `systemctl --user enable model-switcher` 即可自启

### 3.3 跨平台检测逻辑

- **状态**：已结束
- **范围**：`launcher.js` 少量改动
- **内容**：根据 `process.platform` 选择启动参数（Windows 用 `windowsHide`，macOS/Linux 用 `detached` + `unref`）
- **改动量**：约 10 行
- **验收标准**：三大平台上 `node launcher.js` 均能无终端窗口运行

## 阶段四：架构重构

### 4.1 通用配置模型重构

- **状态**：已结束
- **范围**：`config.json` 结构 + `server.js` 读写逻辑
- **内容**：将 `token` 字段统一重命名为 `apiKey`，保持向后兼容读取（`config.apiKey || config.token`）
- **改动量**：约 10 行后端
- **验收标准**：新建配置使用 `apiKey` 字段，旧配置仍可正常读取

### 4.2 插件式 ProviderAdapter

- **状态**：已结束
- **范围**：新增 `providers/` 目录，重构 `server.js`
- **内容**：抽象 ProviderAdapter 接口（`switch`/`verify`/`getFields`），抽取 `claude.js` 和 `opencode.js` 适配器；连通性测试按 `providerType` 抽取到 `testers` 模块
- **改动量**：约 180 行新代码 + server.js 从 447 行精简至 270 行
- **验收标准**：切换、验证、测试功能均通过适配器调用，无硬编码分支

### 4.3 动态表单渲染

- **状态**：已结束
- **范围**：`index.html` + 新增 `presets.json`
- **内容**：Provider 预设数据（URL、模型名、图标）抽取到 `presets.json`，前端通过 `/api/presets` 动态加载；向导 Provider 网格动态生成；名称下拉菜单动态生成
- **改动量**：约 50 行后端 + 100 行前端重构
- **验收标准**：所有预设数据从 presets.json 加载，添加新 Provider 只需修改 presets.json
